# Stretch Task Display Feature

## Overview
The stretch task display feature provides optional advanced challenges to high-performing users during learning sessions. This feature is part of the Learning Session Interface component (Task 22.8).

## Requirements Implemented

### Requirement 14.3: Optional Marking
- Stretch tasks are clearly marked with an "Optional Challenge" badge
- A "Stretch Task" label is displayed
- Explanatory text states: "This is an optional advanced challenge. You can attempt it to push your skills further, or skip it without affecting your progress."

### Requirement 14.4: Complete Action
- Users can attempt the stretch task by clicking the "Attempt Challenge" button
- This focuses the input field, allowing users to respond to the challenge
- Performance is recorded through the normal interaction flow
- Completing stretch tasks does not block progression to the next node

### Requirement 14.5: Skip Action
- Users can skip the stretch task by clicking the "Skip Challenge" button
- Skipping removes the stretch task from view
- Progression continues to the next node without penalty
- No negative impact on mastery score or learning path

## When Stretch Tasks Appear

Stretch tasks are generated by the backend when:
1. Mastery score exceeds 80%
2. Confidence level is "high"

The backend's Adaptive Learning Engine determines when to generate stretch tasks based on these criteria.

## User Flow

### Scenario 1: User Attempts Stretch Task
1. User completes a learning interaction
2. Backend generates stretch task (if criteria met)
3. Stretch task appears in the chat area
4. User clicks "Attempt Challenge"
5. Input field is focused
6. User types their response
7. Response is submitted through normal interaction flow
8. Performance is recorded
9. User continues to next node

### Scenario 2: User Skips Stretch Task
1. User completes a learning interaction
2. Backend generates stretch task (if criteria met)
3. Stretch task appears in the chat area
4. User clicks "Skip Challenge"
5. Stretch task is removed from view
6. User continues to next node without penalty
